## ReasonML vs. JavaScript Cheatsheet

| JavaScript                              | Reason                                                  |
| --------------------------------------- | ------------------------------------------------------- |
| `const x = 5;`                          | `let x = 5;`                                            |
| `var x = y;`                            | No equivalent! üöÄ                                       |
| `let x = 5; x = x + 1;`                 | `let x = ref(5); x := x^ + 1;`                          |
| `const add = a => b => a + b;`          | `let add = (a, b) => a + b;` **(curry by default! üôåüèº)** |
| `'hello' + ' world'`                    | `"hello" ++ " world"` (single quotes are `char`)        |
| No deep equality (recursive compare)    | `a == b`, `a != b`                                      |
| `a == b`                                | No equality with implicit casting _(thankfully)_        |
| `[1, 2, 3]`                             | `[| 1, 2, 3 |]`                                         |
| No immutable list built-in the language | `[1, 2, 3]`                                             |
| `null`, `undefined`                     | `None` (no null bugs, only the type `option()`! üëèüèº)     |

Mutability is always uglier!! üôåüèº

```reason Use your favourite hooks! ü•∞
type action =
  | Tick;

type state = {
  count: int,

[@react.component]
let make = () => {
  let (state, dispatch) = React.useReducer(
    (state, action) =>
      switch (action) {
      | Tick => {count: state.count + 1}
      },
    {count: 0}
  );

  React.useEffect0(() => {
    let timerId = Js.Global.setInterval(() => dispatch(Tick), 1000);
    Some(() => Js.Global.clearInterval(timerId))
  });

};
```
